问题 2 - 回答:Â¶

我们把训练用数据集简称为训练集，测试用数据集简称为测试集。

问题一：将数据集按一定比例分为训练用的数据集和测试用的数据集对学习算法有什么好处？

训练集主要用于训练机器学习模型，测试集内的数据一定程度上模拟了真实世界的数据，主要被用来检测对机器学习模型输入真实世界的数据的时候，机器学习模型进行预测的准确度。

检测机器学习模型的目的是为了评估和改进训练模型的方法，最终训练处更加优秀、更加准确的模型。

将一个数据集按照一定比例划分训练集和测试集的特征是：

  • 两者之间的数据相互独立

  • 两者之间遵循相同的概率分布

问题二：如果用模型已经见过的数据，例如部分训练集数据进行测试，又有什么坏处？

这样做使得相对于训练集数据，用于测试的数据失去了真实世界中数据样本应当具备的独立性，因而无法模拟真实世界中的数据样本对模型的预测能力进行检测和评估，更加无法改进和优化机器学习模型。

更具体地说就是：当使用了所有的原始数据去训练模型,得到的结果很可能是该模型最大程度地拟合（适应）了原始数据，亦即该模型是为了拟合（适应）所有原始数据而存在。当新的样本出现，再使用该模型进行预测，效果可能还不如只使用一部分数据训练的模型。

编程练习3：定义衡量标准Â¶

如果不能对模型的训练和测试的表现进行量化地评估，我们就很难衡量模型的好坏。通常我们会定义一些衡量标准，这些标准可以通过对某些误差或者拟合程度的计算来得到。在这个项目中，你将通过运算决定系数 R2 来量化模型的表现。模型的决定系数是回归分析中十分常用的统计信息，经常被当作衡量模型预测能力好坏的标准。

R2的数值范围从0至1，表示目标变量的预测值和实际值之间的相关程度平方的百分比。一个模型的R2 值为0还不如直接用平均值来预测效果好；而一个R2 值为1的模型则可以对目标变量进行完美的预测。从0至1之间的数值，则表示该模型中目标变量中有百分之多少能够用特征来解释。模型也可能出现负值的R2，这种情况下模型所做预测有时会比直接计算目标变量的平均值差很多。

在下方代码的 performance_metric 函数中，你要实现：

  • 使用 sklearn.metrics 中的 r2_score 来计算 y_true 和 y_predict的R2值，作为对其表现的评判。
  • 将他们的表现评分储存到score变量中。

或

  • (可选) 不使用任何外部库，参考决定系数的定义进行计算，这也可以帮助你更好的理解决定系数在什么情况下等于0或等于1。

问题 8 - 回答：Â¶

  • 什么是K折交叉验证法（k-fold cross-validation）？

回答：在机器学习中，将数据集A分为训练集（training set）B和测试集（test set）C，在样本量不充足的情况下，为了充分利用数据集对算法效果进行测试，将数据集B随机分为k个包，每次将其中一个包作为测试集，剩下k-1个包作为训练集进行训练。

  • GridSearchCV是如何结合交叉验证来完成对最佳参数组合的选择的？

回答：以决策树为例，我们把max_depth作为参数，参数值的可选值是1~10。网格搜索法针对每一个max_depth参数可用值运行一次K折交叉验证法，从而计算出模型在不同max_depth下的得分。

  • GridSearchCV中的cv_results_属性能告诉我们什么？

cv_results_是一个字典对象，字典的key是字符串，value是numpy数组，每一个key代表一个网格搜索运算的结果属性，就像下面的样子：

                            mask = [ True  True False False]...),

在这个练习中，你将需要将所学到的内容整合，使用决策树算法训练一个模型。为了得出的是一个最优模型，你需要使用网格搜索法训练模型，以找到最佳的 'max_depth' 参数。你可以把'max_depth' 参数理解为决策树算法在做出预测前，允许其对数据提出问题的数量。决策树是监督学习算法中的一种。

  3. 定义 'params' 变量: 为 'max_depth' 参数创造一个字典，它的值是从1至10的数组;
  4. 定义 'scoring_fnc' 变量: 使用 sklearn.metrics 中的 make_scorer 创建一个评分函数； 将 ‘performance_metric’ 作为参数传至这个函数中；
  5. 定义 'grid' 变量: 使用 sklearn.model_selection 中的 GridSearchCV 创建一个网格搜索对象；将变量'regressor', 'params', 'scoring_fnc'和 'cross_validator' 作为参数传至这个对象构造函数中；

def fit_model(X, y):
    """ 基于输入数据 [X,y]，利于网格搜索找到最优的决策树模型"""
    
    cross_validator = KFold(n_splits=10)
    
    regressor = DecisionTreeRegressor(random_state=0)

    params = { 'max_depth' : list(range(1, 11)) }
    
    scoring_fnc = make_scorer(performance_metric)

    grid = GridSearchCV(regressor, params, cv=cross_validator, scoring=scoring_fnc)

    # 基于输入数据 [X,y]，进行网格搜索
    grid = grid.fit(X, y)

    # 返回网格搜索后的最优模型
    return grid.best_estimator_

模型健壮性Â¶

一个最优的模型不一定是一个健壮模型。有的时候模型会过于复杂或者过于简单，以致于难以泛化新增添的数据；有的时候模型采用的学习算法并不适用于特定的数据结构；有的时候样本本身可能有太多噪点或样本过少，使得模型无法准确地预测目标变量。这些情况下我们会说模型是欠拟合的。
